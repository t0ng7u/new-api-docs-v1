## üìù Introduction

The OpenAI Audio API provides three main functionalities:

1.  Text-to-Speech (TTS) - Converts text into natural-sounding speech
2.  Speech-to-Text (STT) - Transcribes audio into text
3.  Audio Translation - Translates non-English audio into English text

## üí° Request Examples

### Text-to-Speech ‚úÖ

```bash
curl https://your-newapi-server-address/v1/audio/speech \
  -H "Authorization: Bearer $NEWAPI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tts-1",
    "input": "Hello, world!",
    "voice": "alloy"
  }' \
  --output speech.mp3
```

### Speech-to-Text ‚úÖ

```bash
curl https://your-newapi-server-address/v1/audio/transcriptions \
  -H "Authorization: Bearer $NEWAPI_API_KEY" \
  -H "Content-Type: multipart/form-data" \
  -F file="@/path/to/file/audio.mp3" \
  -F model="whisper-1"
```

**Response Example:**

```json
{
  "text": "Hello, world!"
}
```

### Audio Translation ‚úÖ

```bash
curl https://your-newapi-server-address/v1/audio/translations \

  -H "Content-Type: multipart/form-data" \
  -F file="@/path/to/file/chinese.mp3" \
  -F model="whisper-1"
```

**Response Example:**

```json
{

}
```

## üìÆ Requests

### Endpoints

#### Text-to-Speech

```
POST /v1/audio/speech
```

Converts text into speech.

#### Speech-to-Text

```
POST /v1/audio/transcriptions
```

Transcribes audio into text in the input language.

#### Audio Translation

```
POST /v1/audio/translations
```

Translates audio into English text.

### Authentication Method

Include the following in the request header for API key authentication:

```
Authorization: Bearer $NEWAPI_API_KEY
```

Where `$NEWAPI_API_KEY` is your API key.

### Request Body Parameters

#### Text-to-Speech

##### `model`

-   Type: String
-   Required: Yes
-   Possible values: tts-1, tts-1-hd
-   Description: The TTS model to use

##### `input`

-   Type: String
-   Required: Yes
-   Max length: 4096 characters
-   Description: The text to convert to speech

##### `voice`

-   Type: String
-   Required: Yes
-   Possible values: alloy, echo, fable, onyx, nova, shimmer
-   Description: The voice to use when generating speech

##### `response_format`

-   Type: String
-   Required: No
-   Default: mp3
-   Possible values: mp3, opus, aac, flac, wav, pcm
-   Description: The audio output format

##### `speed`

-   Type: Number
-   Required: No
-   Default: 1.0
-   Range: 0.25 - 4.0
-   Description: The speed of the generated speech

#### Speech-to-Text

##### `file`

-   Type: File
-   Required: Yes
-   Supported formats: flac, mp3, mp4, mpeg, mpga, m4a, ogg, wav, webm
-   Description: The audio file to transcribe

##### `model`

-   Type: String
-   Required: Yes
-   Currently only supports: whisper-1
-   Description: The ID of the model to use

##### `language`

-   Type: String
-   Required: No
-   Format: ISO-639-1 (e.g., "en")
-   Description: The language of the audio, providing it can improve accuracy

##### `prompt`

-   Type: String
-   Required: No
-   Description: Text used to guide the model's style or continue a previous audio segment

##### `response_format`

-   Type: String
-   Required: No
-   Default: json
-   Possible values: json, text, srt, verbose_json, vtt
-   Description: The output format

##### `temperature`

-   Type: Number
-   Required: No
-   Default: 0
-   Range: 0 - 1
-   Description: Sampling temperature, higher values make the output more random

##### `timestamp_granularities`

-   Type: Array
-   Required: No
-   Default: segment
-   Possible values: word, segment
-   Description: The timestamp granularities for the transcription

#### Audio Translation

##### `file`

-   Type: File
-   Required: Yes
-   Supported formats: flac, mp3, mp4, mpeg, mpga, m4a, ogg, wav, webm
-   Description: The audio file to translate

##### `model`

-   Type: String
-   Required: Yes
-   Currently only supports: whisper-1
-   Description: The ID of the model to use

##### `prompt`

-   Type: String
-   Required: No
-   Description: English text used to guide the model's style

##### `response_format`

-   Type: String
-   Required: No
-   Default: json
-   Possible values: json, text, srt, verbose_json, vtt
-   Description: The output format

##### `temperature`

-   Type: Number
-   Required: No
-   Default: 0
-   Range: 0 - 1
-   Description: Sampling temperature, higher values make the output more random

## üì• Responses

### Successful Responses

#### Text-to-Speech

Returns binary audio file content.

#### Speech-to-Text

##### Basic JSON Format

```json
{
  "text": "Transcribed text content"
}
```

##### Detailed JSON Format

```json
{
  "task": "transcribe",
  "language": "english",
  "duration": 8.47,
  "text": "Complete transcribed text",
  "segments": [
    {
      "id": 0,
      "seek": 0,
      "start": 0.0,
      "end": 3.32,
      "text": "Segmented transcribed text",
      "tokens": [50364, 440, 7534],
      "temperature": 0.0,
      "avg_logprob": -0.286,
      "compression_ratio": 1.236,
      "no_speech_prob": 0.009
    }
  ]
}
```

#### Audio Translation

```json
{
  "text": "Translated English text"
}
```

### Error Responses

When a request encounters an issue, the API will return an error response object with an HTTP status code in the 4XX-5XX range.

#### Common Error Status Codes

-   `400 Bad Request`: Invalid request parameters
-   `401 Unauthorized`: Invalid or missing API key
-   `429 Too Many Requests`: API call limit exceeded
-   `500 Internal Server Error`: Internal server error

Error response example:

```json
{
  "error": {
    "message": "File format not supported",
    "type": "invalid_request_error",
    "param": "file",
    "code": "invalid_file_format"
  }
}
```
