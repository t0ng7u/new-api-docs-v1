## üìÆ Request

### Endpoint

```
POST /v1/rerank
```

### Authentication Method

Include the following in the request header for API key authentication:

```
Authorization: Bearer $NEWAPI_API_KEY
```

Where `$NEWAPI_API_KEY` is your API key.

### Request Body Parameters

#### `model`

- Type: String
- Required: No
- Default: jina-reranker-v2-base-multilingual
- Description: The reranking model to use

#### `query`

- Type: String
- Required: Yes
- Description: The query text used to sort documents by relevance

#### `top_n`

- Type: Integer
- Required: No
- Default: Unlimited
- Description: Returns the top N reranked documents

#### `documents`

- Type: Array of Strings
- Required: Yes
- Description: The list of documents to rerank
- Constraint: The length of each document should not exceed the model's maximum token limit

## üì• Response

### Successful Response

#### `results`

- Type: Array
- Description: The list of reranked documents
- Properties:
  - `document`: An object containing the document text
  - `index`: The index of the document in the original list
  - `relevance_score`: The relevance score (between 0-1)

#### `usage`

- Type: Object
- Description: Token usage statistics
- Properties:
  - `prompt_tokens`: The number of tokens used for the prompt
  - `completion_tokens`: The number of tokens used for completion
  - `total_tokens`: The total number of tokens
  - `prompt_tokens_details`: Detailed prompt token information
    - `cached_tokens`: The number of cached tokens
    - `audio_tokens`: The number of audio tokens
  - `completion_tokens_details`: Detailed completion token information
    - `reasoning_tokens`: The number of reasoning tokens
    - `audio_tokens`: The number of audio tokens
    - `accepted_prediction_tokens`: The number of accepted prediction tokens
    - `rejected_prediction_tokens`: The number of rejected prediction tokens

### Error Response

When there is an issue with the request, the API will return an error response:

- `400 Bad Request`: Invalid request parameters
- `401 Unauthorized`: Invalid or missing API key
- `429 Too Many Requests`: Request frequency exceeded
- `500 Internal Server Error`: Internal server error

## üí° Best Practices

### Query Optimization Suggestions

1. Use clear and specific query text
2. Avoid overly broad or vague queries
3. Ensure the query uses the same language style as the documents

### Document Processing Suggestions

1. Keep document length moderate, not exceeding model limits
2. Ensure document content is complete and meaningful
3. Can include multilingual documents, the model supports cross-language matching

### Performance Optimization

1. Set the `top_n` parameter appropriately to reduce unnecessary computation
2. For a large number of documents, consider batch processing
3. Results of common queries can be cached

### Multilingual Support

This model supports document reranking in multiple languages, including but not limited to:

- English
- Chinese
- German
- Spanish
- Japanese
- French

No need to specify language parameters; the model automatically identifies and processes content in different languages.
