<Callout type="info" title="Feature Description">
The API prefix is uniformly http(s)://`<your-domain>`
</Callout>

    HTTPS should be used in production environments to ensure authentication tokens. HTTP is only recommended for development environments.

    System configuration management with the highest privileges, accessible only by Root users. Includes features such as global parameter configuration, model ratio reset, and console setting migration. Configuration updates include strict dependency validation logic.

### Get Global Configuration

- **Interface Name**: Get Global Configuration
- **HTTP Method**: GET
- **Path**: `/api/option/`
- **Authentication Requirement**: Root
- **Function Description**: Retrieves all global configuration options for the system, filtering sensitive information such as Tokens, Secrets, Keys, etc.
  üí° Request Example:

```
const response = await fetch('/api/option/', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_root_token',
    'New-Api-User': 'your_user_id'
  }
});
const data = await response.json();
```

‚úÖ Successful Response Example:

```
{
  "success": true,
  "message": "",
  "data": [
    {
      "key": "SystemName",
      "value": "New API"
    },
    {
      "key": "DisplayInCurrencyEnabled",
      "value": "true"
    },
    {
      "key": "QuotaPerUnit",
      "value": "500000"
    }
  ]
}
```

‚ùó Failed Response Example:

```

  "success": false,

üßæ Field Description:

`data` (Array): List of configuration items option.goÔºö15-18

- `key` (String): Configuration item key name

### Update Global Configuration

- **Interface Name**: Update Global Configuration
- **HTTP Method**: PUT
- **Path**: `/api/option/`
- **Authentication Requirement**: Root
- **Function Description**: Updates a single global configuration item, including configuration validation and dependency checks

üí° Request Example:

```
const response = await fetch('/api/option/', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_root_token',
    'New-Api-User': 'your_user_id'
  },
  body: JSON.stringify({
    key: "SystemName",
    value: "My New API System"
  })
});
const data = await response.json();
```

‚úÖ Successful Response Example:

```
{
  "success": true,
  "message": "Configuration updated successfully"
}
```

‚ùó Failed Response Example:

```
{
  "success": false,
  "message": "Unable to enable GitHub OAuth, please fill in GitHub Client Id and GitHub Client Secret first!"
}
```

üßæ Field Description:

- `key` (String): Configuration item key name, required option.goÔºö39-42
- `value` (Any Type): Configuration item value, supports boolean, number, string, and other types option.goÔºö54-63

### Reset Model Ratios

- **Interface Name**: Reset Model Ratios
- **HTTP Method**: POST
- **Path**: `/api/option/rest_model_ratio`
- **Authentication Requirement**: Root
- **Function Description**: Resets all model ratio configurations to their default values, used for bulk resetting model pricing

üí° Request Example:

```

  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_root_token',
    'New-Api-User': 'your_user_id'
  }
});
const data = await response.json();
```

‚úÖ Successful Response Example:

```
{
  "success": true,
  "message": "Model ratios reset successfully"
}
```

‚ùó Failed Response Example:

```

  "success": false,

üßæ Field Description:

No request parameters, all model ratio configurations will be reset after execution

### Migrate Old Console Settings

- **Interface Name**: Migrate Old Console Settings
- **HTTP Method**: POST
- **Path**: `/api/option/migrate_console_setting`
- **Authentication Requirement**: Root
- **Function Description**: Migrates old version console settings to the new configuration format, including API information, announcements, FAQ, etc.

üí° Request Example:

```
const response = await fetch('/api/option/migrate_console_setting', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_root_token',
    'New-Api-User': 'your_user_id'
  }
});
const data = await response.json();
```

‚úÖ Successful Response Example:

```
{
  "success": true,
  "message": "migrated"
}
```

‚ùó Failed Response Example:

```
{
  "success": false,
  "message": "Migration failed"
}
```

üßæ Field Description:

- No request parameters
- Migration content includes:
  - `ApiInfo` ‚Üí `console_setting.api_info`
  - `Announcements` ‚Üí `console_setting.announcements`
  - `FAQ` ‚Üí `console_setting.faq`
  - `UptimeKumaUrl/UptimeKumaSlug` ‚Üí `console_setting.uptime_kuma_groups`
